What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment (or Delivery).
CI automatically checks code changes when developers push commits.
It can run validations/tests to catch issues early and keep the project stable.
CD then deploys the approved version automatically to the target environment.
This reduces manual work and makes releases faster and more reliable.

My Pipeline

After I push to the main branch, GitHub Actions starts a workflow run.
It checks required files, uploads the site artifact, then deploys to GitHub Pages.
When the workflow succeeds, the live website updates automatically.
